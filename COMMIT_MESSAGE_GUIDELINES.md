# Commit Message Guidelines

Use Conventional Commit style for consistency:

- feat: add a new feature
- fix: fix a bug
- docs: documentation updates
- refactor: code changes without feature or fix
- test: add or update tests
- chore: maintenance and tooling

Format:

<type>(optional-scope): short summary

Examples:

- feat(lms-api): add publish course endpoint
- fix(simplelms): correct enrollment progress validation
- docs(readme): update backend-first contribution scope

Rules:

- Keep summary short and clear.
- Use imperative mood.
- Avoid unrelated changes in one commit.
