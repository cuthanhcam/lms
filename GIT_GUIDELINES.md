# Git Guidelines

## Branching

- Base branch: default branch (typically main)
- Feature branch naming:
  - feature/[short-description]
  - bugfix/[short-description]
  - chore/[short-description]
  - docs/[short-description]

## Pull Requests

- Keep PR scope small and focused.
- Link issue when applicable.
- Include local build and test evidence.
- Resolve review comments before merge.

## Recommended Flow

1. Pull latest default branch.
2. Create a topic branch.
3. Commit in small, logical steps.
4. Rebase or merge default branch as needed.
5. Open PR using the project template.
